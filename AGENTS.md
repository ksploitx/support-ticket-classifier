# Project and Agent Rules

## Coding Style
- Keep the project extremely simple and hand-coded.
- Avoid unnecessary "extra junk" or heavy boilerplate. 

## Command Execution & Git
- Do not run execution commands (like running the app) automatically without asking. Provide the command to the user, explain what it does, and let the user execute it.
- **NEVER** commit on your own. When it is time to commit, provide the `git add` and `git commit` commands to the user so they can do it.

## Phase Completion
- At the end of every major phase or feature, provide a clear summary of what was built.
- Include explicit instructions on how the user can check, test, and run the new code.
